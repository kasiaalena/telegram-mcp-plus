# Тесты производительности telegram-mcp-plus

## Метрики

Для каждого теста замеряем:
- **Latency** — время от вызова tool до получения ответа (мс)
- **Memory** — потребление RAM процессом (MB)
- **CPU** — загрузка процессора (%)

## Процедура запуска тестов производительности

```bash
# 1. Замер времени отклика
python -m pytest tests/test_performance.py -v --tb=short

# 2. Замер памяти
python -c "
import tracemalloc
tracemalloc.start()
# ... выполнить серию запросов ...
current, peak = tracemalloc.get_traced_memory()
print(f'Current: {current / 1024 / 1024:.1f} MB')
print(f'Peak: {peak / 1024 / 1024:.1f} MB')
"

# 3. Замер потребления процесса
# Запустить сервер, в другом терминале:
```

## Инструменты

- **pytest-benchmark** — автоматический замер latency
- **tracemalloc** — отслеживание памяти Python
- **psutil** — системные метрики (CPU, RAM)
- **time.perf_counter** — точный замер времени
