# Тесты безопасности telegram-mcp-plus

## Категория 1: Защита сессии

| # | Тест | Что проверяем | Критерий прохождения |
|---|------|---------------|---------------------|
| S1.1 | Права доступа к файлу сессии | Файл .session имеет права 600 (только владелец) | `stat -f %Lp session.session` → 600 |
| S1.2 | Сессия не в stdout | При работе сервера сессионные данные не попадают в stdout | Перехват stdout не содержит auth_key |
| S1.3 | Сессия не в логах | Логи не содержат auth_key, session string | Grep по логам = 0 совпадений |
| S1.4 | Директория сессии защищена | ~/.telegram-mcp-plus/ имеет права 700 | Только владелец может войти |
| S1.5 | Сессия не передаётся в MCP-ответах | Ни один tool не возвращает данные сессии | Проверка всех ответов tools |

## Категория 2: Защита credentials

| # | Тест | Что проверяем | Критерий прохождения |
|---|------|---------------|---------------------|
| S2.1 | API Hash не в логах | TG_API_HASH не выводится ни в какие логи | Grep по stderr = 0 |
| S2.2 | API Hash не в ответах | Ни один tool не возвращает api_hash | Проверка всех JSON-ответов |
| S2.3 | Телефон маскирован | В логах телефон показывается как +7***...89 | Проверка формата в stderr |
| S2.4 | Нет hardcoded secrets | В исходниках нет захардкоженных токенов/ключей | grep по паттернам секретов = 0 |

## Категория 3: Валидация входных данных

| # | Тест | Вход | Критерий прохождения |
|---|------|------|---------------------|
| S3.1 | SQL injection в name | name="'; DROP TABLE--" | Ошибка "Dialog not found", нет SQL-выполнения |
| S3.2 | Path traversal в name | name="../../etc/passwd" | Ошибка "Dialog not found" |
| S3.3 | XSS в text | text="<script>alert(1)</script>" | Текст отправляется как есть (plain text), не исполняется |
| S3.4 | Очень длинный name | name = "a" * 10000 | Ошибка валидации, не OOM |
| S3.5 | Очень длинный text | text = "a" * 100000 | Ошибка: превышен лимит длины |
| S3.6 | Unicode-атака | name с zero-width chars, RTL override | Корректная обработка или понятная ошибка |
| S3.7 | Null bytes | name="user\x00name" | Ошибка валидации |
| S3.8 | Отрицательные числа | limit=-1, folder_id=-1 | Ошибка валидации |
| S3.9 | Числовое переполнение | offset_id=99999999999999999 | Ошибка или корректная обработка |

## Категория 4: Защита от утечки данных

| # | Тест | Что проверяем | Критерий прохождения |
|---|------|---------------|---------------------|
| S4.1 | Нет лишних полей в tg_me | Ответ содержит только id, first_name, last_name, username | Нет phone, auth_key, session |
| S4.2 | Сообщения не кешируются на диск | После запроса tg_dialog нет temp-файлов с сообщениями | Проверка /tmp и рабочей директории |
| S4.3 | Ошибки не раскрывают внутренности | Стектрейсы не попадают в MCP-ответы | isError ответы содержат только user-friendly текст |
| S4.4 | Нет утечки entity metadata | Ответы не содержат access_hash, raw Telegram объекты | Только публичные поля |

## Категория 5: Сетевая безопасность

| # | Тест | Что проверяем | Критерий прохождения |
|---|------|---------------|---------------------|
| S5.1 | Только MTProto | Сервер не открывает HTTP/WebSocket порты | netstat/lsof показывает 0 listening портов |
| S5.2 | Нет локальных сетевых сервисов | stdio-only, никаких TCP-серверов | Проверка при запуске |
| S5.3 | TLS для Telegram | Соединение с Telegram через encrypted MTProto | Telethon использует это по умолчанию |

## Категория 6: Ошибки авторизации

| # | Тест | Что проверяем | Критерий прохождения |
|---|------|---------------|---------------------|
| S6.1 | Brute-force кода | 5 неверных кодов подряд | FloodWait обработан, понятное сообщение |
| S6.2 | Неверный 2FA пароль | Неправильный password | Понятная ошибка, не crash |
| S6.3 | Отмена авторизации | Ctrl+C во время ввода кода | Graceful exit, нет битой сессии |
| S6.4 | Невалидные env vars | TG_APP_ID="abc" | Понятная ошибка при старте |

## Процедура запуска тестов безопасности

```bash
# 1. Проверка прав файлов
stat -f "%Lp %N" ~/.telegram-mcp-plus/*

# 2. Проверка отсутствия секретов в логах
telegram-mcp-plus 2>stderr.log
grep -iE "(api_hash|auth_key|session|password)" stderr.log

# 3. Проверка отсутствия открытых портов
lsof -i -P -n | grep telegram-mcp

# 4. Проверка валидации входных данных — через MCP tool calls
# (автоматизировано в test_security.py)
```
